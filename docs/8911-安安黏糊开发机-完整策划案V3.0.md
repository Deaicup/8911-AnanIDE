## 项目定位

基于 GitHub 成熟开源 IDE（Eclipse Theia）二次开发的二次元风格独立 AI 开发软件，内置可视化代码编辑器与原生 MCP 协议终端，以原创虚拟偶像「安安」为核心人格载体，实现 IDE 与 CLI 深度协同的全自动化开发流程，全程零版权风险。

**一句话定义**：一个会让安安陪你写代码、自动编排 MCP 服务做安全扫描/测试/部署、全程不跳出软件的二次元 AI 开发机。

## 一、项目背景与选型依据

### 1.1 行业痛点

当前主流 AI 开发工具普遍存在三大问题：
- 跨 IDE 与终端操作上下文断裂
- MCP 协议配置复杂门槛高
- 工具交互冰冷缺乏情绪价值，开发者在长时间编码过程中容易产生疲劳感，高危命令误操作风险高

### 1.2 开源基底选型

| 选型项目 | GitHub 仓库 | 开源协议 | 选型优势 |
|---------|------------|---------|---------|
| **Eclipse Theia（确定采用）** | https://github.com/eclipse-theia/theia | MIT | VS Code 架构的官方开源实现，自带完整编辑、终端、插件系统，生态成熟 |
| 备选 Zed | https://github.com/zed-industries/zed | Apache 2.0 | Rust 高性能内核，启动速度极快，适合轻量化定制 |

**最终选型理由**：Theia 是 TypeScript 全栈，扩展系统成熟，原生支持 Electron 打包，与本项目技术栈完全契合。Zed 虽性能更强但 Rust 内核对二次开发门槛高。

### 1.3 版权合规说明（已修订）

| 组件 | 协议 | 商用条件 |
|------|------|---------|
| Eclipse Theia 基底 | MIT | 免费商用，零版权风险 |
| 动画引擎 Inochi2D | BSD-2-Clause | 免费商用，无扩展性 APP 限制 |
| 动画兜底 PixiJS | MIT | 免费商用 |
| 安安形象素材 | 原创绘制 | 自有著作权，零第三方风险 |

**重要修订**：原 V1.0 策划声称使用 Live2D，经核实 Live2D Cubism SDK 对"可扩展性应用程序"不享受免费豁免，正式发行前必须签署出版许可协议并付费。本版本已替换为 Inochi2D（BSD-2-Clause），彻底消除该风险。

## 二、产品核心架构

### 2.1 三层架构

```
┌─────────────────────────────────────────────┐
│  表现层：二次元 UI + 安安 Live2D 交互层       │
│  (anan-ui: 主题/图标/动画/调试面板)          │
├─────────────────────────────────────────────┤
│  中间层：安安人格中枢 + MCP 协议网关          │
│  (anan-core + anan-mcp)                     │
├─────────────────────────────────────────────┤
│  底层：Eclipse Theia 内核（复用，不改源码）   │
│  (编辑/文件管理/调试/插件系统)               │
└─────────────────────────────────────────────┘
```

### 2.2 核心模块划分

#### 模块 1：可视化二次元 IDE 编辑区

- 替换默认主题系统，内置 3 套安安专属萌系配色方案（粉系/蓝系/暗夜）
- 代码行号旁搭载动态安安表情，根据代码质量自动切换状态（正常/开心/思考/警告/错误）
- 侧边栏文件树采用二次元图标设计，安安自动为修改文件贴彩色标记贴纸
- 调试面板变量状态用萌系卡片展示，断点命中时安安主动弹窗提示异常值

#### 模块 2：原生 MCP CLI 终端

- 终端默认提示符替换为「安安✨ ~」，原生支持 MCP 协议自动发现
- 无需手动配置，启动软件自动扫描本地 Workbuddy、codebase-memory-mcp 等所有 MCP 服务
- 支持自然语言直接转 MCP 指令，隐藏复杂协议配置细节
- 高危 MCP 操作自动触发黏糊式二次确认，避免误删等风险
- MCP 任务流可视化追踪，搭配安安专属进度动画，执行结果自动同步回编辑器

#### 模块 3：安安人格中枢（双环境共享核心）

- 统一本地知识图谱，全程记忆代码编辑状态、命令历史、MCP 调用结果
- 智能多 MCP 服务编排，自动串联安全扫描、代码校验、测试生成等全流程任务
- 状态联动反馈系统，根据编码时长自动触发休息提醒、情绪互动
- 全本地隐私架构，所有数据不流出本地，满足企业级代码安全要求

## 五、各模块技术方案

### 5.1 二次元 UI 替换（anan-ui）

**MVP 阶段**：
- 通过 Theia 的 `ColorRegistry` 注册 3 套安安配色（粉系/蓝系/暗夜）
- 替换文件树图标为二次元风格 SVG
- 编辑器行号旁渲染 PixiJS Canvas，显示安安表情（5 种状态：正常/开心/思考/警告/错误）
- 代码质量通过 TypeScript Language Server 的 diagnostic 信息驱动表情切换

**迭代阶段**：
- PixiJS 替换为 Inochi2D WASM
- 从 PSD 分层素材创建 Inochi2D Puppet 模型
- 实现骨骼动画、物理模拟（头发/衣服摆动）
- 断点命中时安安弹窗（Inochi2D 驱动）

**技术要点**：
- Theia 的 UI 定制通过 Extension 实现，不改内核源码
- 配色用 `ColorContribution` 接口注册
- 图标用 `IconThemeContribution` 注册自定义图标主题
- 安安动画层是一个固定定位的 Overlay Widget，用 Theia 的 `Widget` API 挂载

### 5.2 MCP 终端扩展（anan-mcp）

**自动发现机制**：
1. 启动时扫描 `~/.workbuddy/mcp.json`、`~/.config/mcp/servers.json`、环境变量 `MCP_SERVERS`
2. 扫描常见 MCP 服务端口（localhost:56834 等）
3. 对每个发现的 MCP 服务，用 `@modelcontextprotocol/sdk` 发 `initialize` 请求验证可用性
4. 结果写入 SQLite，UI 显示已连接服务列表

**高危操作确认**：
- 维护危险命令黑名单：`rm`、`del`、`format`、`dd`、`mkfs`、`> /dev/`、`shutdown`、`Drop`、`truncate`、`writeFile(覆盖)`、`deleteFile`
- 黑名单匹配到 → 安安弹窗 + 3 秒倒计时 + 红色高亮显示命令
- 倒计时结束前不执行，用户可随时取消

**NL→MCP 转换（迭代阶段）**：
- 本地运行 Ollama + Qwen2.5-7B（需 8GB+ 显存或 16GB+ 内存）
- Prompt 模板：用户输入 + 可用 MCP 工具列表 → 输出结构化 JSON（tool_name + parameters）
- MVP 阶段用命令模板替代：`anan scan` → `mcp.security-scan.run()`，`anan test` → `mcp.workbuddy.generate-tests()`
- 模板系统支持自定义，用户可在配置中添加新模板

### 5.3 安安人格中枢（anan-core）

**知识图谱存储**：
- SQLite 表结构：`events(id, type, content, project, timestamp, metadata)`
- 记录：文件编辑历史、终端命令、MCP 调用结果、用户交互
- 查询接口：按项目/时间/类型检索，支持模糊搜索

**多 MCP 编排**：
- 编排规则用 JSON 定义，存储在 `~/.anan/orchestrations/`
- 示例：安全检查编排 = `security-scan` → `code-lint` → `test-generate` → 汇总报告
- 编排引擎按 DAG 依赖顺序执行，支持并行和串行
- 每步结果写入 SQLite，失败自动重试 1 次

**状态联动**：
- 编码 2 小时 → 安安提醒休息
- 连续报错 3 次 → 安安建议换个思路
- 成功编译 → 安安庆祝动画
- 触发规则用配置文件管理，用户可自定义

### 5.4 防呆设计（anan-core/safety）

统一安全模块，被所有其他模块调用，所有模块调用 Safety 模块做前置检查，不直接执行用户输入。

**输入层防护**：
- 所有输入框做长度/类型/路径校验
- 文件打开拦截超大文件和二进制文件
- 终端危险命令默认拦截 + 安安弹窗确认

**状态层防护**：
- 编辑中关闭自动保存 + 下次启动恢复
- MCP 执行中断网优雅降级
- 配置文件损坏自动恢复默认值
- 禁止多开（单实例锁定）

**操作层防护**：
- 删除走回收站不直接删
- 高危 MCP 操作 3 秒倒计时确认
- 所有按钮防抖

**资源层防护**：
- 内存超阈值自动释放
- 窗口最小尺寸限制
- 磁盘满预警

## 六、数据持久化

| 数据类型 | 存储位置 | 格式 |
|---------|---------|------|
| 用户配置 | `~/.anan/config.json` | JSON |
| 知识图谱/事件历史 | `~/.anan/data.db` | SQLite |
| MCP 编排规则 | `~/.anan/orchestrations/*.json` | JSON |
| 安安素材 | `app://assets/anan/` | 打包内置 |
| 用户自定义素材 | `~/.anan/assets/` | 用户管理 |
| 自动保存文件 | `~/.anan/autosave/<project>/` | 原文件格式 |
| 配置备份 | `~/.anan/config.backup.json` | JSON（每次启动自动备份） |

**崩溃恢复**：
- 每次文件修改后 5 秒延迟写入 autosave 目录
- 启动时检查 autosave，有未恢复文件则弹窗提示
- 用户选择恢复或丢弃，选择后清理 autosave

## 七、打包方案

- 安装包大小控制在 200MB 以内（Theia + Electron 基础约 150MB）

### 10.1 人员

### 10.2 头盖骨职责

承担全部代码开发工作，按阶段递进：

| 阶段 | 核心任务 |
|------|---------|
| POC（1-2周） | Theia fork 构建、主题替换验证、Inochi2D WASM 加载 Demo |
| MVP（3-8周） | 二次元 UI 全套替换、PixiJS 安安动画、MCP 终端扩展、高危操作确认弹窗、防呆设计、Electron 打包 |
| 迭代（9-18周） | Inochi2D 完整集成、安安人格中枢、多 MCP 编排引擎、NL→MCP 转换 |

**防呆设计要求**：输入校验、危险命令拦截、自动保存/崩溃恢复、按钮防抖、内存上限、单实例锁定、配置损坏恢复、回收站机制、高危 MCP 3秒倒计时确认。

**开发心法**：每写一个功能先问 "feiyu 会怎么搞坏这个？"

### 10.3 feiyu 职责

专注做一件事：**打包成 exe 后，用"不按套路"的方式搞坏软件**。不写代码，不搭框架，纯黑盒。把软件当成一个完全不懂技术的用户在用，什么操作离谱就试什么。

**测试方向**：

| 类别 | 具体场景 |
|------|---------|
| 暴力操作 | 狂点按钮、缩窗口到最小、编辑中强杀进程、同时开多个实例、往编辑器拖 exe/zip/iso |
| 非常规输入 | 终端粘贴 `rm -rf /`、文件名用特殊字符或超长中文、打开 500MB 文件、MCP 执行中拔网线、手动改坏配置文件再启动 |

**验收红线**：
- **P0（阻断发布）**：崩溃、数据丢失、安全漏洞
- **P1（需修复）**：有兜底提示但体验差
- **P2（记录下版本优化）**：小瑕疵

**流程**：feiyu 输出缺陷清单 → 头盖骨修复 → feiyu 回归验证 → 签字放行

### 10.4 协作约定

- 头盖骨提的每个模块打包后，feiyu 必须做一轮防呆测试
- feiyu 测出 P0 问题，头盖骨立即修复，修复后 feiyu 回归验证才能放行
- 每周五下午做一次联调演示

## 十一、典型使用流程

1. 软件启动，安安主动打招呼，后台 1 秒内自动完成所有本地 MCP 服务对接，零手动配置
2. 在可视化编辑器中编写完新接口代码，安安自动提示是否启动全流程 MCP 检查
3. 切换内置终端输入自然语言需求，安安自动转换为 MCP 指令流，依次调用安全扫描、代码校验、Workbuddy 测试生成服务
4. 所有任务执行完成，结果自动同步回编辑器，优化建议直接插入对应代码行
5. 终端输入提交部署指令，安安自动生成提交信息并完成上线，全程无跨软件跳转

| 迭代 | 第 9-18 周 | Inochi2D 集成、人格中枢、MCP 编排、NL→MCP | 每个模块打包后做一轮防呆测试 |
| 生态 | 持续运营 | 开放插件接口、素材市场 | 社区反馈跟踪 |

**排期说明**：原 V1.0 排期 4+6 周过于激进，已修订为 2+6+10 周（POC+MVP+迭代），2 人团队下为合理上限。
